## 7.1 系统信息

### 4 个 API 速查

| 函数 | 头文件 | 拿什么 | 重要度 |
|------|-------|--------|-------|
| `uname(&buf)` | `<sys/utsname.h>` | OS 名 / 主机名 / 内核版本 / 架构 | ★★ |
| `sysinfo(&buf)` | `<sys/sysinfo.h>` | 内存 / 负载 / uptime | ★ |
| `gethostname(buf, len)` | `<unistd.h>` | 主机名 | ★ |
| **`sysconf(_SC_xxx)`** | `<unistd.h>` | **系统配置常量** | **★★★★** |

### ⭐ sysconf 是最有用的一个

```c
long page_size = sysconf(_SC_PAGESIZE);   // 通常 4096
long clk_tck   = sysconf(_SC_CLK_TCK);    // 时钟节拍数
long max_open  = sysconf(_SC_OPEN_MAX);   // 进程最大 fd 数
```

**为什么记 `_SC_PAGESIZE`?**
- 后面学**驱动 mmap** 必须按 PAGESIZE 对齐
- 申请 **DMA 缓冲区**要 PAGESIZE 对齐
- 不能写死 4096,某些 ARM 平台是 16384

## 7.2 时间日期

### 概念区分(面试可能问)

- **GMT** = 格林威治标准时间(天文计算)
- **UTC** = 协调世界时(原子钟,**现代标准**)
- **CST** = 中国标准时间 = UTC + 8
- **Unix 时间戳** = 从 1970-01-01 00:00:00 UTC 至今的秒数(time_t)
- **jiffies** = Linux 内核的"时钟节拍"(HZ=100 时,1 jiffy = 10ms)

### API 速查(知道有,用查 man)

| 函数 | 精度 | 用途 |
|------|------|------|
| `time(NULL)` | 秒 | 拿当前 Unix 时间戳 |
| `gettimeofday(&tv, NULL)` | **微秒** | 高精度时间戳(tv_sec + tv_usec) |
| `clock_gettime(CLOCK_MONOTONIC, &ts)` | **纳秒** | 测时间间隔最准(单调时钟) |
| `ctime(&t)` | - | time_t → "Mon Feb 22 17:10:46 2021" |
| `localtime_r(&t, &tm)` | - | time_t → struct tm(本地时间) |
| `gmtime_r(&t, &tm)` | - | time_t → struct tm(UTC 时间) |
| `mktime(&tm)` | - | struct tm → time_t |
| `strftime(buf, n, fmt, &tm)` | - | struct tm → 任意格式字符串 |

### 唯一要记的模板:测程序运行时间

```c
struct timeval start, end;
gettimeofday(&start, NULL);
/* ... 要测的代码 ... */
gettimeofday(&end, NULL);

long us = (end.tv_sec - start.tv_sec) * 1000000
        + (end.tv_usec - start.tv_usec);
printf("Elapsed: %ld us\n", us);
```

### 其他全跳过

`localtime/gmtime/asctime/strftime` 这套"日期格式化"5 年用不上一次,**用到查 `man 3 strftime`**。

## 7.4 随机数(常识)

```c
#include <stdlib.h>
srand(time(NULL));      // 用当前时间播种(不然每次跑结果一样)
int r = rand() % 100;   // 0~99 的伪随机数
```

**铁律**: 必须 `srand(time(NULL))` 播种一次,否则程序每次跑 rand() 返回值都一样。

**加密用途别用 rand**,用 `/dev/urandom` 或 `getrandom()`。

## 7.5 休眠(三种精度)

| 函数 | 头文件 | 精度 | 单位 |
|------|-------|------|------|
| `sleep(s)` | `<unistd.h>` | 秒 | unsigned int |
| `usleep(us)` | `<unistd.h>` | 微秒 | useconds_t |
| `nanosleep(&req, &rem)` | `<time.h>` | **纳秒** | struct timespec |

```c
// 睡 3 秒,3 种写法
sleep(3);
usleep(3 * 1000 * 1000);
struct timespec req = { .tv_sec = 3, .tv_nsec = 0 };
nanosleep(&req, NULL);
```

**关键认知**: sleep 让 CPU **真休息**,不是 for 循环空转(那样会跑满 CPU)。嵌入式低功耗设计的基础。

**面试可能问**: "怎么实现一个 10ms 的精确延时?"
**答**: nanosleep 是用户态最优解;内核态用 hrtimer;裸机用 SysTick 中断。

## 7.6 堆内存

### 基础三件套

```c
void *malloc(size_t size);              // 分配,内容未定义
void *calloc(size_t nmemb, size_t size);// 分配 + 清零
void free(void *ptr);                    // 释放
```

| | malloc | calloc |
|---|-------|--------|
| 参数 | 总字节数 | 元素数 × 单个大小 |
| 清零 | ❌ | ✅ |
| 等价 | `calloc(n, s)` = `memset(malloc(n*s), 0, n*s)` | |

### 铁律(面试常考)

| 规则 | 后果 |
|------|------|
| **malloc 和 free 必须成对** | 不 free → 内存泄漏 |
| **不要 free 已 free 的指针** | double free → 程序崩 |
| **free 后指针置 NULL** | 防止 use-after-free |
| **不要 free 栈/全局变量** | malloc 之外的内存 free 会崩 |
| **超出分配范围读写** | 缓冲区溢出,玄学 bug |

```c
char *p = malloc(100);
if (p == NULL) { perror("malloc"); return -1; }
/* ... use p ... */
free(p);
p = NULL;     // 好习惯
```

### ⭐ 对齐内存分配(嵌入式必用)

普通 malloc 默认对齐:32 位 8 字节,64 位 16 字节。
但 **DMA 缓冲区**、**O_DIRECT 直接 I/O** 需要更大对齐(通常 4096)。

5 个对齐分配函数(**只记 posix_memalign 就行**,其他都是历史包袱):

```c
#include <stdlib.h>

int posix_memalign(void **memptr, size_t alignment, size_t size);
```

```c
int *base;
posix_memalign((void **)&base, 256, 1024);   // 256 字节对齐的 1024 字节
free(base);    // 注意:还是用 free 释放
```

**参数约束**:
- `alignment` 必须是 **2 的幂**(64/128/256/512/1024/2048/4096...)
- 且 **>= sizeof(void *)**(64 位系统 >= 8)
- 返回 **0 成功**(注意!不是返回指针),失败返回 errno 值

**其他 4 个**(知道存在即可,**已弃用,别在新代码用**):
- `aligned_alloc(alignment, size)` — C11 标准
- `memalign(alignment, size)` — 老接口
- `valloc(size)` — 按 page 对齐
- `pvalloc(size)` — 按 page 对齐 + size 也对齐

### 嵌入式应用场景

```c
// DMA 缓冲区,4KB 对齐
static char *dma_buf;
posix_memalign((void **)&dma_buf, 4096, 4096);

// O_DIRECT 直接 I/O 缓冲区
static char *direct_buf;
posix_memalign((void **)&direct_buf, getpagesize(), 8192);
```

后面学驱动会大量遇到 `__attribute__((aligned(N)))` 和 `dma_alloc_coherent`,**本质都是"对齐的内存分配"**。

## 7.7 proc 文件系统 ⭐⭐⭐(本章灵魂)

### 概念

`/proc` 是**虚拟文件系统**(不在磁盘,内核动态生成),把**进程信息 + 系统信息**暴露成"文件",**用户态用普通 read 就能读**。

### 常用 /proc 节点

| 路径 | 内容 |
|------|------|
| `/proc/cpuinfo` | CPU 型号 / 主频 / 核心数 |
| `/proc/meminfo` | 内存总量 / 可用 / Buffer / Cache |
| `/proc/version` | 内核版本 + GCC 版本 |
| `/proc/uptime` | 系统运行时间 |
| `/proc/loadavg` | 1/5/15 分钟负载 |
| `/proc/interrupts` | 中断统计 |
| `/proc/iomem` | 内存映射 I/O |
| `/proc/modules` | 已加载内核模块 |
| `/proc/<PID>/` | 进程 PID 的信息(每进程一个目录) |
| `/proc/<PID>/cmdline` | 进程启动命令行 |
| `/proc/<PID>/status` | 进程状态(内存/线程/调度) |
| `/proc/<PID>/fd/` | 进程打开的 fd 列表 |

### 读取方法(就用普通文件 I/O)

```c
int fd = open("/proc/version", O_RDONLY);
char buf[512] = {0};
read(fd, buf, sizeof(buf));
puts(buf);
close(fd);
```

**等价命令行**:
```bash
cat /proc/version
```

### ⭐ 为什么 proc 对学驱动这么重要?

1. **写驱动可以暴露 /proc 节点** — 用户态用 `cat /proc/mydevice` 就能看驱动状态
2. **调试时大量用** — 看 `/proc/interrupts` 知道驱动中断号、统计;`/proc/iomem` 知道寄存器映射地址
3. **不需要写专门工具** — top / free / ps / lsof 这些命令底层都是 `cat /proc/xxx` 然后解析

### 工具命令对照

| 命令 | 实际读什么 |
|------|---------|
| `uname -r` | `/proc/sys/kernel/osrelease` |
| `free -m` | `/proc/meminfo` |
| `top` | `/proc/[pid]/stat` 每个进程 |
| `lscpu` | `/proc/cpuinfo` |
| `lsmod` | `/proc/modules` |

**面试可能问**: "top 命令怎么实现的?"
**答**: 读 `/proc` 下每个 PID 目录的 stat 文件解析。

## 踩坑

### 1. localtime 返回值不能跨调用使用

```c
struct tm *t1 = localtime(&time1);   // 静态缓冲区
struct tm *t2 = localtime(&time2);   // ⚠ 同一个缓冲区,t1 被覆盖
```
**修复**: 用 `localtime_r(&time1, &my_tm)`,自己提供 struct tm。

### 2. rand() 不播种每次跑结果一样

```c
int r = rand();  // ⚠ 没 srand,每次跑都是同一个数
```
**修复**: 程序开头 `srand(time(NULL));`

### 3. posix_memalign 返回值是 errno 不是 -1

```c
int ret = posix_memalign(&p, 4096, 1024);
if (ret != 0) {                        // ✓
    fprintf(stderr, "%s\n", strerror(ret));
}
// 别写 if (ret == -1) — 错的
```

### 4. sleep 可能被信号打断提前返回

```c
sleep(10);   // 如果中途收到信号,可能 5 秒就返回了
```
**修复**: 看返回值(sleep 返回剩余秒数),或者用 nanosleep 配合 rem 重试。
