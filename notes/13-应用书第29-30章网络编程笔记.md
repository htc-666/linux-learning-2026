## 一、socket 是什么(一句话理解)

```
socket = "能跨主机的文件描述符"
  文件: open()→fd→read/write→close()
  socket: socket()→套接字fd→recv/send→close()
→ 像读写文件一样收发网络数据,底层TCP/IP(握手/重传)被socket封装,你不用管
```
- IP 定位"哪台机器",端口 定位"机器上哪个程序" → IP+端口 唯一锁定一个服务
- 🔵 服务器(硬件机器) ≠ 服务端(角色:listen+accept等连接的程序)。谁等连接谁是服务端,哪怕只是块开发板。

## 二、核心 API 详解(逐个拆)

> 头文件: `#include <sys/socket.h>` `#include <netinet/in.h>` `#include <arpa/inet.h>`

### 1. socket() — 建套接字
```c
int socket(int domain, int type, int protocol);
```
- **domain（协议族）**：`AF_INET` = IPv4（最常用）；`AF_INET6` = IPv6；`AF_UNIX` = 本机进程间
- **type（套接字类型）**：`SOCK_STREAM` = TCP（可靠、面向连接、字节流）；`SOCK_DGRAM` = UDP（不可靠、无连接、报文）
- **protocol（协议）**：填 `0` = 自动选默认（SOCK_STREAM 默认 TCP，SOCK_DGRAM 默认 UDP）
- **返回值**：成功 = 套接字描述符（≥0，类似 fd）；失败 = `-1`（设 errno）
```c
int sockfd = socket(AF_INET, SOCK_STREAM, 0);   // 建一个 TCP 套接字
if (sockfd < 0) { perror("socket"); exit(-1); }
```

### 2. bind() — 绑 IP+端口（服务器用）
```c
int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```
- **sockfd**：socket() 返回的套接字
- **addr**：指向地址结构体（实际传 `struct sockaddr_in`，要强转 `(struct sockaddr *)`）
- **addrlen**：地址结构体大小 `sizeof(addr)`
- **返回值**：成功 `0`；失败 `-1`
```c
bind(sockfd, (struct sockaddr *)&server_addr, sizeof(server_addr));
```
> ⚠️ 端口被占用会 bind 失败（"Address already in use"）—— 上次程序没退干净就会这样，换端口或等会儿。

### 3. listen() — 进入监听（服务器用）
```c
int listen(int sockfd, int backlog);
```
- **sockfd**：已 bind 的套接字
- **backlog**：等待连接队列的最大长度（同时有多个客户端来连、还没 accept 时，最多排多少个）。一般填 50/128
- **返回值**：成功 `0`；失败 `-1`
```c
listen(sockfd, 50);
```
> listen 后服务器进入"监听态"，但还没接受任何连接 —— 真正接受要靠 accept。

### 4. accept() — 接受连接，返回新套接字（服务器用）⭐
```c
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```
- **sockfd**：监听套接字
- **addr**：传出参数，存**客户端的地址**（IP+端口）；不关心就传 `NULL`
- **addrlen**：传入传出，传入时 = `sizeof(addr)`，传出时 = 实际地址长度；传 `NULL` 配合 addr=NULL
- **返回值**：成功 = **新套接字 connfd**（代表这条连接，收发用它）；失败 `-1`
```c
socklen_t len = sizeof(client_addr);
int connfd = accept(sockfd, (struct sockaddr *)&client_addr, &len);
```
> ⭐ 核心：**accept 阻塞等连接，返回的 connfd 才是和这个客户的"专线"**。
> sockfd 继续监听新客户，connfd 负责和这个客户 recv/send。
> ⚠️ addrlen 是 **socklen_t \*（指针）**，要传地址 `&len`，且 len 要先初始化成 `sizeof`。

### 5. connect() — 连接服务器（客户端用）
```c
int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```
- **sockfd**：客户端的套接字
- **addr**：**服务器的**地址（要填服务器 IP + 端口）
- **addrlen**：`sizeof(addr)`
- **返回值**：成功 `0`；失败 `-1`
```c
connect(sockfd, (struct sockaddr *)&server_addr, sizeof(server_addr));
```
> connect 对 TCP 会触发**三次握手**；返回 0 = 握手成功、连接建立。
> ⚠️ 服务器没启动 / IP端口错 → connect 失败（"Connection refused"）。

### 6. recv() / send() — 收发数据（双方用）
```c
ssize_t recv(int sockfd, void *buf, size_t len, int flags);
ssize_t send(int sockfd, const void *buf, size_t len, int flags);
```
- **sockfd**：连接套接字（服务器用 **connfd**，客户端用 sockfd）
- **buf**：recv = 接收缓冲区；send = 要发的数据
- **len**：recv = 缓冲区大小；send = **要发的实际长度（用 `strlen(buf)`，别用 sizeof）**
- **flags**：一般填 `0`（= 和 read/write 一样）；偶尔用 MSG_DONTWAIT/MSG_PEEK
- **返回值**：
  - `recv`：`>0` 收到的字节数；`==0` **对方关闭连接**；`-1` 出错
  - `send`：返回实际发送的字节数；`-1` 出错
```c
ret = recv(connfd, buf, sizeof(buf), 0);
ret = send(sockfd, buf, strlen(buf), 0);
```
> 💡 套接字也是 fd，所以 **read/write 也能用**（`read(connfd,buf,len)` / `write(connfd,buf,len)`），效果同 flags=0 的 recv/send。recv/send 只是多了 flags 参数。

### 7. close() — 关闭
```c
int close(int fd);
```
- 关套接字、释放资源。对 TCP 会触发**四次挥手**断开连接。
- 服务器要关两个：`close(connfd)`（这条连接）+ `close(sockfd)`（监听套接字）。

## 地址结构体 + 转换函数（配合上面用）

### struct sockaddr_in（实际填的地址结构）
```c
struct sockaddr_in {
    sa_family_t    sin_family;   // 协议族, 填 AF_INET
    in_port_t      sin_port;     // 端口号(网络字节序, 用 htons)
    struct in_addr sin_addr;     // IP地址(sin_addr.s_addr)
    unsigned char  sin_zero[8];  // 填充, 不用管(memset清零即可)
};
```
> 函数参数要的是通用的 `struct sockaddr *`，所以传的时候**强转** `(struct sockaddr *)&addr`。

### 字节序转换
```c
uint16_t htons(uint16_t);   // host→network short(2字节): 端口用
uint32_t htonl(uint32_t);   // host→network long(4字节):  IP用
uint16_t ntohs(uint16_t);   // network→host: 收到的端口转回来展示用
uint32_t ntohl(uint32_t);
```

### IP 字符串 ↔ 二进制
```c
int  inet_pton(int af, const char *src, void *dst);   // "192.168.1.1"字符串 → 二进制(填进sin_addr)
const char *inet_ntop(int af, const void *src, char *dst, socklen_t size);  // 二进制 → 字符串(展示用)
// af 填 AF_INET; pton成功返回1
```
```c
inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);           // 客户端填服务器IP
char ipstr[16];
inet_ntop(AF_INET, &cli_addr.sin_addr, ipstr, sizeof(ipstr));  // 把客户端IP转成字符串打印
```

```
服务器: socket → bind → listen → accept → recv/send → close
客户端: socket →           connect → send/recv → close
                    ↑客户端没有bind/listen/accept,多了connect
```

| 函数 | 谁用 | 作用 |
|------|------|------|
| `socket(AF_INET, SOCK_STREAM, 0)` | 双方 | 建套接字。SOCK_STREAM=TCP, SOCK_DGRAM=UDP |
| `bind(fd, &addr, len)` | 服务器 | 绑 IP+端口(挂出固定号码) |
| `listen(fd, backlog)` | 服务器 | 进入监听,backlog=等待队列上限 |
| `accept(fd, &cliaddr, &len)` | 服务器 | 阻塞等连接,⭐**返回【新套接字connfd】代表这条连接** |
| `connect(fd, &addr, len)` | 客户端 | 连服务器,TCP在此触发三次握手 |
| `recv(fd,buf,len,0)/send(fd,buf,len,0)` | 双方 | 收/发。等价 read/write,flags一般0 |
| `close(fd)` | 双方 | 关闭 |

### ⭐ accept 返回的两个套接字(面试点)
```
sockfd(socket返回)  = 监听套接字 = 前台总机,只管"接新客户",一直守着
connfd(accept返回)  = 连接套接字 = 专属分机,跟"这一个客户"的recv/send全走它
→ bind/listen/accept 用 sockfd;  recv/send/close这条连接 用 connfd
→ 两个分工:一个守门接新客户,一个陪老客户聊 → 服务器才能同时应付多客户
⚠️ 新手最爱错: 拿 sockfd 去 recv/send → 错!收发必须用 connfd
```

## 三、地址结构体 + 字节序

```c
struct sockaddr_in addr = {0};
addr.sin_family      = AF_INET;
addr.sin_port        = htons(8888);          // 端口→网络字节序
addr.sin_addr.s_addr = htonl(INADDR_ANY);    // 服务器:监听本机所有网卡(0.0.0.0)
// 客户端填服务器IP:
inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);
```

### 字节序(大小端)= 你八股的知识点,落地就是这几个
```
htons/htonl: host→network  主机字节序→网络字节序(大端)
ntohs/ntohl: network→host  收数据要展示时转回来
s=short(2字节,端口)  l=long(4字节,IP)
→ 端口用 htons, IP用 htonl
为什么: 网络规定大端;你电脑可能小端;不转两台机器数字读错
```

### IP 转换: inet_pton(字符串→二进制),不是赋值!
```c
inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);   // ✅ "翻译官":字符串IP→32位二进制
addr.sin_addr.s_addr = "127.0.0.1";                // ❌ 字符串不能直接赋给数字!
// INADDR_ANY 能直接赋(htonl),是因为它本身是数字(0),不是字符串
```

## 四、recv 返回值三态(必记,踩过坑)
```
ret > 0  → 收到 ret 字节数据
ret == 0 → 【对方关闭了连接】(TCP正常断开) ← 必须 break,否则死循环!
ret < 0  → 出错
```

## 五、🔴 实战踩坑全记录(自己写 server.c + client.c 踩的)

```
【宏/变量】
  ① #define SERVER_PORT 8888;  → 宏不能带分号(展开成htons(8888;)语法错)
  ② if(socket < 0)  → 写成函数名了,应是 if(sockfd < 0)
  ③ socklen_t 写成 socket_t  → 类型不存在,编译报错
  ④ 客户端 inet_pton 里用了 server_addr,但变量名是 client_addr → 未定义

【recv/send】
  ⑤ recv 只判 ret<0,漏了 ret==0(对方断开) → 客户端断开会死循环打印空数据
  ⑥ send 用 sizeof(buf)=512 → 每次发满512字节(含\0);应 strlen(buf) 发实际长度

【大括号错位】(经典)
  ⑦ 把"打印数据"塞进 else if(ret==0) 块、还在break后 = 死代码,永不执行
     把 close/return 塞进 for 循环体内 → 收到一条数据就close+return退出
  教训: { } 对齐缩进; VS Code点括号看配对

【两个fgets】
  ⑧ 加"带NULL检查的fgets"时没删原来的fgets → 读了两次
     → 要按两次回车,第一次输入被覆盖丢弃,只有第二次发出去
  教训: 加代码看清是"替换"还是"追加"

【运行环境】
  ⑨ VS Code的"运行按钮/Code Runner"不给stdin → fgets读EOF → 疯狂刷屏
     → 要在【真终端手动敲 ./client】,别点运行按钮
  ⑩ 改完代码不生效 → 没存盘/没重新gcc → 三步: Ctrl+S存盘 → gcc编译 → 运行
```

## 六、🔵 TCP 是字节流(粘包)—— 实战见到的真概念

```
你以为: 发一句=收一句(一一对应)
实际上: TCP是【字节流】,不保证消息边界
  → 连着快发: 服务器一次recv可能把好几条一起收 = 粘包
  → 一条也可能被拆成两次recv
→ "发N次 ≠ recv N次"
解决: 实际项目自己定协议(加长度头/分隔符)切分消息
注意: 但TCP是【可靠】的——数据不会丢、不会乱序、不会损坏,只是边界不保留
```
> 这正是你八股"粘包/队头阻塞",现在亲眼见到实物。

## 七、完整代码骨架(参考)

**server.c**
```c
sockfd = socket(AF_INET, SOCK_STREAM, 0);
addr.sin_family=AF_INET; addr.sin_addr.s_addr=htonl(INADDR_ANY); addr.sin_port=htons(8888);
bind(sockfd, (struct sockaddr*)&addr, sizeof(addr));
listen(sockfd, 50);
connfd = accept(sockfd, (struct sockaddr*)&cliaddr, &len);   // 返回新套接字
for(;;){
    ret = recv(connfd, buf, sizeof(buf), 0);
    if(ret<0){...break;} else if(ret==0){断开;break;}
    printf("from client: %s\n", buf);
    if(!strncmp(buf,"exit",4)) break;
}
close(connfd); close(sockfd);
```

**client.c**
```c
sockfd = socket(AF_INET, SOCK_STREAM, 0);
addr.sin_family=AF_INET; addr.sin_port=htons(8888);
inet_pton(AF_INET, "127.0.0.1", &addr.sin_addr);
connect(sockfd, (struct sockaddr*)&addr, sizeof(addr));
for(;;){
    if(fgets(buf,sizeof(buf),stdin)==NULL) break;
    send(sockfd, buf, strlen(buf), 0);
    if(!strncmp(buf,"exit",4)) break;
}
close(sockfd);
```

## 一句话总结
> **socket = 跨主机的fd。服务器 socket→bind→listen→accept(得connfd)→recv/send;客户端 socket→connect→send/recv。**
> **收发用 accept 返回的 connfd(不是监听的sockfd);端口htons/IP htonl转字节序;IP字符串用inet_pton转。**
> **TCP是可靠的字节流(粘包,要自己切消息)。ch29理论见我的网络八股。**
