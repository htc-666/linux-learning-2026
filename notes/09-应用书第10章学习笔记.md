# 第10章 进程间通信 IPC(概览章 · 复习卡)

> 本章是纯概念目录章,无代码。核心是认识 IPC 的分类 + 各方式特点。
> 详细实现书里没展开(直接跳到第11章线程)。

## IPC 四大分类(本章主线)

```
1. UNIX IPC     → 管道(pipe)、FIFO、信号(signal)
2. System V IPC → 消息队列、信号量、共享内存   (老,AT&T)
3. POSIX IPC    → 消息队列、信号量、共享内存   (新,更规范,推荐)
4. Socket IPC   → 套接字(唯一能跨主机)
```
> 记忆点: System V 和 POSIX 都各有"消息队列/信号量/共享内存"三件套,
>         POSIX 是更现代的版本。

## 6 大方式对比 ⭐(面试核心,背这张表)

| 方式 | 一句话 | 关键点 |
|------|--------|--------|
| 匿名管道 pipe | 内存单向通道 | **只能亲缘进程**(父子),半双工 |
| 命名管道 FIFO | 有文件名的管道 | **任意进程**可用,半双工 |
| 信号 signal | 异步通知 | 信息量极少(就一个信号编号) |
| 消息队列 | 带类型的消息链表 | 可按类型取,随内核持续 |
| 共享内存 | 多进程映射同一内存 | ⭐**最快**(无拷贝),需配信号量同步 |
| 信号量 semaphore | 计数器 | ⭐**做同步/互斥的,不传数据** |
| Socket | 套接字 | ⭐**唯一能跨主机**(网络) |

## 八股必背 3 连问

```
Q: 哪个最快?
A: 共享内存(直接读写同一块内存,省去内核拷贝);
   但它不带同步,要配信号量一起用。

Q: 匿名管道 vs FIFO?
A: 匿名管道只能亲缘进程; FIFO 有文件名,任意进程都能用。

Q: 信号量是干嘛的?
A: 同步/互斥(控制访问),不是传数据! (最易答错)
```

## 一句话总结

> **IPC 分 UNIX/System V/POSIX/Socket 四类;**
> **记住:共享内存最快(配信号量)、信号量是同步不是传数据、Socket 能跨主机。**
