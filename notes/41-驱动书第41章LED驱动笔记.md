## 41.1 Linux 下 LED 灯驱动原理 ⭐⭐⭐⭐⭐

第41章的核心可以拆成两层：

| 层次 | 内容 | 你现在掌握情况 |
|---|---|---|
| Linux 驱动框架 | `file_operations`、`register_chrdev`、`copy_from_user`、`module_init/module_exit` | 已经能看懂 |
| I.MX6ULL 寄存器控制 | `CCM_CCGR1`、`IOMUXC`、`GPIO1_GDIR`、`GPIO1_DR`、`readl/writel` | 先理解流程，不用死背每个位 |

本章 LED 驱动 = **第40章字符设备框架 + 裸机式 GPIO 寄存器配置**。

### 41.1.1 地址映射：为什么要 `ioremap`？

裸机时可以直接操作物理地址：

```c
*(volatile unsigned int *)0x020E0068 = 5;
```

但 Linux 启动后 MMU 已开启，CPU 访问的是虚拟地址，不应该直接访问外设物理地址。因此驱动要先用 `ioremap()` 把寄存器物理地址映射成内核虚拟地址。

```c
#define SW_MUX_GPIO1_IO03_BASE  (0x020E0068)
static void __iomem *SW_MUX_GPIO1_IO03;

SW_MUX_GPIO1_IO03 = ioremap(SW_MUX_GPIO1_IO03_BASE, 4);
```

卸载驱动时要取消映射：

```c
iounmap(SW_MUX_GPIO1_IO03);
```

🔵 记忆：`ioremap` 是 **物理地址 → 内核虚拟地址**；`iounmap` 是释放映射。

### 41.1.2 I/O 内存访问函数

ARM 没有独立 I/O 端口空间，外设寄存器以内存映射方式访问，所以叫 I/O 内存。

常用访问函数：

| 函数 | 位宽 | 作用 |
|---|---|---|
| `readb()` / `writeb()` | 8 bit | 字节访问 |
| `readw()` / `writew()` | 16 bit | 半字访问 |
| `readl()` / `writel()` | 32 bit | 字访问 |

I.MX6ULL 寄存器是 32 位，所以本章使用：

```c
val = readl(GPIO1_DR);
writel(val, GPIO1_DR);
```

## 41.2 硬件原理图分析 ⭐⭐⭐⭐

ALPHA 板的用户 LED 接在：

```text
GPIO1_IO03
```

并且是低电平点亮：

```text
GPIO1_IO03 输出 0 → LED 亮
GPIO1_IO03 输出 1 → LED 灭
```

本章涉及 5 个寄存器：

| 寄存器 | 地址 | 作用 |
|---|---|---|
| `CCM_CCGR1` | `0x020C406C` | 开 GPIO1 时钟 |
| `SW_MUX_GPIO1_IO03` | `0x020E0068` | 设置引脚复用功能 |
| `SW_PAD_GPIO1_IO03` | `0x020E02F4` | 设置引脚电气属性 |
| `GPIO1_GDIR` | `0x0209C004` | 设置 GPIO 输入/输出方向 |
| `GPIO1_DR` | `0x0209C000` | 设置 GPIO 输出电平 |

🔵 你这次的问题主要集中在这里：驱动框架能看懂，但 GPIO/时钟寄存器不清楚。这个很正常，因为这里本质是 I.MX6ULL 裸机外设知识，不是 Linux 字符设备框架本身。

## GPIO 点灯五步模板 ⭐⭐⭐⭐⭐

以后看到类似 GPIO 代码，先不要被寄存器名吓住，抓住这 5 步：

```text
1. 开 GPIO 模块时钟
2. 设置引脚复用为 GPIO 功能
3. 设置 PAD 电气属性
4. 设置 GPIO 方向为输出
5. 写 GPIO 数据寄存器控制高低电平
```

### 1. 开 GPIO1 时钟

```c
val = readl(IMX6U_CCM_CCGR1);
val &= ~(3 << 26);
val |=  (3 << 26);
writel(val, IMX6U_CCM_CCGR1);
```

含义：

```text
CCM_CCGR1 是时钟门控寄存器
bit[27:26] 控制 GPIO1 时钟
写 0b11 表示开启
```

🔵 外设工作前一般都要先开时钟。不开时钟，寄存器配置可能无效，外设不工作。

### 2. 设置引脚复用功能

```c
writel(5, SW_MUX_GPIO1_IO03);
```

I.MX6ULL 的一个物理引脚可以复用成多个功能，例如 UART、I2C、PWM、GPIO 等。

这里写 `5` 表示选择 `ALT5`，也就是：

```text
GPIO1_IO03
```

### 3. 设置 PAD 电气属性

```c
writel(0x10B0, SW_PAD_GPIO1_IO03);
```

这一步配置的是引脚电气属性，例如：

```text
上下拉
驱动能力
速度
压摆率
开漏/非开漏
```

🔵 现在不用死抠 `0x10B0` 每一位。后面第45章 `pinctrl` 会把这类配置放进设备树统一管理。

### 4. 设置 GPIO1_IO03 为输出

```c
val = readl(GPIO1_GDIR);
val &= ~(1 << 3);
val |=  (1 << 3);
writel(val, GPIO1_GDIR);
```

`GDIR` 是方向寄存器：

```text
0 = 输入
1 = 输出
```

`1 << 3` 表示操作 GPIO1 的第 3 位，也就是 GPIO1_IO03。

### 5. 写数据寄存器控制亮灭

```c
val = readl(GPIO1_DR);
val &= ~(1 << 3);
writel(val, GPIO1_DR);
```

GPIO1_IO03 输出低电平，LED 亮。

```c
val = readl(GPIO1_DR);
val |= (1 << 3);
writel(val, GPIO1_DR);
```

GPIO1_IO03 输出高电平，LED 灭。

## 41.3 实验程序编写 ⭐⭐⭐⭐⭐

本章驱动代码结构：

```text
寄存器物理地址宏定义
        ↓
ioremap 映射寄存器
        ↓
初始化 GPIO1_IO03
        ↓
注册字符设备 led
        ↓
应用层 write(fd, &ledstat, 1)
        ↓
驱动 led_write()
        ↓
copy_from_user()
        ↓
led_switch()
        ↓
修改 GPIO1_DR 点灯/灭灯
```

### `led_switch()`

```c
void led_switch(u8 sta)
{
    u32 val = 0;

    if (sta == LEDON) {
        val = readl(GPIO1_DR);
        val &= ~(1 << 3);
        writel(val, GPIO1_DR);
    } else if (sta == LEDOFF) {
        val = readl(GPIO1_DR);
        val |= (1 << 3);
        writel(val, GPIO1_DR);
    }
}
```

注意：

```text
LEDON  → bit3 清 0 → 低电平 → LED 亮
LEDOFF → bit3 置 1 → 高电平 → LED 灭
```

### `led_write()`

```c
retvalue = copy_from_user(databuf, buf, cnt);
ledstat = databuf[0];

if (ledstat == LEDON) {
    led_switch(LEDON);
} else if (ledstat == LEDOFF) {
    led_switch(LEDOFF);
}
```

用户态 APP 向 `/dev/led` 写 1 或 0，驱动根据写入值控制 LED。

## 本章个人理解与疑问记录

### 1. 驱动框架能看懂，但 GPIO/时钟不清楚，正常吗？

正常。

第41章有两套知识：

```text
Linux 字符设备框架
I.MX6ULL 裸机寄存器配置
```

你现在能看懂驱动框架，说明主线已经对了。GPIO/时钟这部分先按“五步模板”理解即可。

### 2. 这些寄存器地址需要背吗？

不用背。

要知道它们负责什么：

```text
CCM_CCGR1           开时钟
SW_MUX_GPIO1_IO03   选复用功能
SW_PAD_GPIO1_IO03   配电气属性
GPIO1_GDIR          配方向
GPIO1_DR            写高低电平
```

### 3. 为什么后面还要学设备树和 pinctrl？

因为第41章这种写法是“裸机式 Linux 驱动”：驱动里直接写死寄存器物理地址。

后面更正规写法是：

```text
设备树描述硬件资源
pinctrl 配置复用和 PAD
gpio 子系统操作 GPIO
驱动不再手写寄存器地址
```

## 面试速查

**1. Linux 驱动里为什么不能直接访问物理地址？**  
因为 MMU 开启后 CPU 访问虚拟地址，驱动要用 `ioremap()` 把物理地址映射成内核虚拟地址。

**2. `readl/writel` 是干什么的？**  
对 I/O 内存进行 32 位读写，常用于访问外设寄存器。

**3. GPIO 点灯一般需要哪些步骤？**  
开时钟、设置引脚复用、配置 PAD 属性、设置方向、写数据寄存器。

**4. 为什么 LEDON 反而是清零？**  
因为 ALPHA 板 LED 是低电平点亮，GPIO 输出 0 时 LED 亮。

**5. 第41章和第40章最大区别？**  
第40章是虚拟字符设备；第41章在字符设备框架中加入真实硬件寄存器控制。
