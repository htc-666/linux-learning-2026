# 第11章 多线程 pthread ⭐⭐⭐⭐(嵌入式面试重头戏)

> 通用规则:
> - 头文件: `#include <pthread.h>`
> - 编译: `gcc xxx.c -o xxx -lpthread`(必须加 -lpthread,放最后)
> - pthread 函数: **成功返回 0,失败【直接返回错误码】**(不是 -1+errno)
>   → 报错用 `strerror(ret)`,不是 `perror`

---

## 11.1 线程概述 ⭐⭐⭐(概念,面试主线)

### 什么是线程
- 一个进程内可以有多个执行流,每个执行流就是一个线程
- 程序启动时就有一个**主线程**(执行 main());其余靠 `pthread_create` 创建
- 线程是 **CPU 调度的基本单位**;进程是 **资源分配的基本单位**

### 🔵 进程 vs 线程(八股最高频,必背)
| | 进程 | 线程 |
|---|------|------|
| 是什么 | 资源分配单位 | CPU调度单位 |
| 地址空间 | 各自独立 | **同进程内共享** |
| 切换开销 | 大(换页表/TLB) | 小 |
| 通信方式 | 要IPC | 直接读全局变量(但要加锁) |
| 健壮性 | 一个崩溃不影响别的 | **一个崩溃整个进程挂** |

### 🔵 线程 共享什么 / 独占什么(整个11→12章的逻辑链)
```
共享: 代码段、全局变量、堆(malloc)、文件描述符、信号处理方式、工作目录
独占: 栈(局部变量)、寄存器、线程ID、errno、信号掩码
```
> 逻辑链: 全局/堆共享 → 多线程同时改会冲突(竞态) → 需要锁/线程安全 → 引出 11.10 + 第12章

### 🔵 Linux 线程的底层实现(面试高频,书没讲)
- Linux 内核**没有独立的"线程"概念**——线程和进程都用 `task_struct` 表示
- 创建线程用 `clone()` 系统调用(fork 底层也是 clone),区别只在【共享了哪些资源】
- 所以 Linux 线程又叫 **LWP**(Light Weight Process,轻量级进程)
- 现代 Linux 用 **NPTL** 库,1:1 模型:1 个用户线程 ↔ 1 个内核调度实体
- 命令: `ps -eLf` 或 `top -H` 能看到线程(LWP 列)

## 11.2 线程 ID ⭐⭐

```c
pthread_t pthread_self(void);                  // 获取自己的TID
int pthread_equal(pthread_t t1, pthread_t t2); // 比较TID,相等返回非0
```
- `pthread_t` 是**不透明类型**(Linux 下实为 `unsigned long`),**不能直接用 `==` 比**,要用 `pthread_equal`
- 区别清楚: `getpid()` 是进程 ID(同进程所有线程相同);`pthread_self()` 是线程 ID(各不同)

## 11.3 创建线程 pthread_create ⭐⭐⭐⭐(核心)

```c
int pthread_create(pthread_t *thread,             // 出参: 新线程ID
                   const pthread_attr_t *attr,    // 属性,一般 NULL(默认)
                   void *(*start_routine)(void*), // 线程函数(签名固定)
                   void *arg);                    // 传给线程函数的实参
```

**PDF 示例 11.3.1**:
```c
static void *new_thread_start(void *arg) {
    printf("子线程: PID<%d> TID<%lu>\n", getpid(), pthread_self());
    return (void *)0;
}
int main(void) {
    pthread_t tid;
    int ret = pthread_create(&tid, NULL, new_thread_start, NULL);
    if (ret) { fprintf(stderr, "Error: %s\n", strerror(ret)); exit(-1); }
    printf("主线程: PID<%d> TID<%lu>\n", getpid(), pthread_self());
    sleep(1);   // 不睡的话主线程exit会带走子线程
    exit(0);
}
```

要点:
- 线程函数签名**固定**为 `void *func(void *)`
- 主子线程 **PID 相同**(同进程)、**TID 不同**

### 🔴 demo 1 踩坑实录 (`ch11/01_thread_basic.c`)
- **坑**: 编译 `gcc 01_thread_basic.c -o 01_thread_basic` 报 `undefined reference to pthread_create`
- **原因**: pthread 函数在单独的 libpthread 库里,不在 libc 里,必须 `-lpthread`
- **修复**: `gcc 01_thread_basic.c -o 01_thread_basic -lpthread`(`-lpthread` 放在源文件之后)
- **现象**:
  ```
  主线程 TID = 140687460878144
  子线程 TID = 140687452382976    ← TID 不同
  子线程返回 49                    ← 7×7 通过 join 带回
  ```

## 11.4 线程退出 pthread_exit ⭐⭐⭐

线程结束的 **3 种方式**:
```
1. 线程函数里 return            ← 最常用
2. pthread_exit(retval)         ← 可在任意深度的子函数里结束本线程
3. 被别的线程 pthread_cancel    ← 见 11.6
⚠️ 任一线程调 exit()/_exit() → 【整个进程】结束(所有线程一起死)
```

```c
void pthread_exit(void *retval);   // retval 由 pthread_join 接收
```

🔵 易错: return/pthread_exit 的返回值**不能是局部变量的地址**(线程栈回收后变野指针)。要么返回值本身强转 `(void*)`,要么返回堆/全局地址。

---

## 11.5 回收线程 pthread_join ⭐⭐⭐⭐(核心,类比 wait)

```c
int pthread_join(pthread_t thread, void **retval);  // 阻塞等thread结束,并取其返回值
```

**PDF 示例 11.5.1**:
```c
static void *new_thread_start(void *arg) {
    sleep(2);
    pthread_exit((void *)10);              // 返回 10
}
int main(void) {
    pthread_t tid; void *tret;
    pthread_create(&tid, NULL, new_thread_start, NULL);
    pthread_join(tid, &tret);              // 阻塞 2 秒
    printf("code=%ld\n", (long)tret);      // 打印 10
    exit(0);
}
```
要点:
- 不 join(也不 detach)已结束的线程 → **僵尸线程,资源泄漏**(类比僵尸进程)
- 🔵 vs wait:join 能**指定**等哪个线程、顺序随意;wait 只能等"任意一个"子进程

### 🔴 demo 2 踩坑实录:传参的经典竞态 (`ch11/02_arg_trap.c`)

**注**:这个 demo 实际验的是 11.3 的"怎么把数据正确交给线程",归在这里因为它**展示了竞态的雏形**(为 11.10/第12章铺垫)。

**功能**:循环创建 5 个线程,对比两种传参方式。

```c
// ❌ 错误版: 5 个线程共享同一个 i
for (int i = 0; i < 5; i++)
    pthread_create(&tid[i], NULL, worker, &i);

// ✅ 正确版: 每个线程拿独立、不被改的地址
static int nums[5] = {0,1,2,3,4};
for (int i = 0; i < 5; i++)
    pthread_create(&tid[i], NULL, worker, &nums[i]);
```

**现象**(每次跑都不一样):
```
错误版第1次: 4 4 5 5 1     ← 重复4、重复5、出现5、缺0/2/3
错误版第2次: 2 1 5 4 5     ← 重复5、出现5、缺0/3
正确版:      0-4 齐全,仅顺序乱(线程调度无序,正常)
```

**三个症状全齐**:
1. **出现 5**:循环结束时 `i==5`(因 i<5 不成立才退出),慢一拍的线程读到了 5
2. **重复**:多个线程同一时刻读 `i`,读到一样的值
3. **缺号**:`i` 短暂等于某值的瞬间没有线程在读 → 丢失

**本质**:5 个线程共享同一个 `i`,线程读 `*arg` 的瞬间 `i` 早被主线程改掉。
**面试要点**:这就是**竞态条件**——结果不确定、不可复现。

## 11.6 取消线程 pthread_cancel ⭐⭐

```c
int pthread_cancel(pthread_t thread);
```

三个相关概念:

**① 取消状态** `pthread_setcancelstate(state, oldstate)`
```c
pthread_setcancelstate(PTHREAD_CANCEL_DISABLE, NULL); // 线程"免疫"取消
// PTHREAD_CANCEL_ENABLE(默认) / DISABLE
```

**② 取消类型** `pthread_setcanceltype(type, oldtype)`
```c
// PTHREAD_CANCEL_DEFERRED(默认): 只在"取消点"才响应
// PTHREAD_CANCEL_ASYNCHRONOUS: 立即响应
```

**③ 取消点(cancellation point)** ⭐关键现象
```
取消点 = 一批阻塞系统调用(sleep/read/write/printf/wait/accept...,PDF 有整张表)
默认 DEFERRED 模式下,线程必须走到取消点才响应取消请求
→ 纯计算死循环 for(;;){} 里没取消点 → cancel 不动!
解决: 手动加一个取消点 pthread_testcancel();
```

### 🔴 demo 4 踩坑实录 (`ch11/04_cancel.c`)

**对比两次运行**:
```
空循环 for(;;){}              → 卡死, Ctrl+C 强退       ❌取消不掉
加 pthread_testcancel()       → "子线程已被取消"        ✅
```

**判断逻辑**(易和 demo 3 搞混):
```c
ret1 = pthread_join(tid, &a);
if (a == PTHREAD_CANCELED)     // ✅ 查【线程退出值 a】,不是 join 返回值
    printf("子线程已被取消\n");
```
> 对比记忆:
> - **demo 3** 关心"join 成没成功" → 查 **join 的返回值**
> - **demo 4** 关心"线程是被取消还是正常结束" → 查 **线程的退出值 a**

**踩的坑**:
1. 头文件漏写 `#include <pthread.h>`(写了两次 `<string.h>`)
2. 一开始用 `if(ret1)` 判断,但 ret1 是 join 成功与否,跟"是否被取消"无关 → 改成查 `a`

**延伸**:如果空循环里有 `printf`,**其实也能被取消**——因为 printf 内部调 `write`,而 `write` 是取消点。

## 11.7 分离线程 pthread_detach ⭐⭐⭐("不用我收尸")

```c
int pthread_detach(pthread_t thread);
```
- detach 后线程结束时**自动回收**,**不能再 join**
- 自分离写法: `pthread_detach(pthread_self());`
- **结论(必记)**: 每个线程要么 join、要么 detach,**二选一**,否则资源泄漏

### 🔴 demo 3 踩坑实录 (`ch11/03_detach.c`)

**预期现象**:线程自己 detach 后,主线程 join → 报 `Invalid argument`(EINVAL)。

**踩的坑**(都关于"两个返回值搞混了"):

```c
ret1 = pthread_join(tid, &a);
//     ↑ 第①个: join 自己的返回值(int 错误码) ← 本 demo 要查的
//                        ↑ 第②个: 线程的退出值(void*) → 存进 a
```

具体踩了 3 次:
1. **看错对象**:最初 `if(a){...}` 判断的是线程返回值,而线程 return NULL → if 永远不成立 → 看不到失败
2. **`strerror(&a)`**:`strerror` 要的是 `int` 错误码,不是指针
3. **变量声明 ≠ 赋值**:`int ret1;` 后直接 `if(ret1)`,读到未初始化垃圾值 → 结果靠运气
   - 修复:`ret1 = pthread_join(tid, &a);` 把返回值接住

**最终现象**:
```
我已分离
失败,原因是Invalid argument        ✅
```

## 11.8 线程清理函数 ⭐⭐(类比 atexit)

```c
void pthread_cleanup_push(void (*routine)(void*), void *arg); // 注册清理函数
void pthread_cleanup_pop(int execute);                         // execute非0则执行一次
```
- 线程退出(`pthread_exit` 或被 cancel)时,**自动**依次调用注册的清理函数(常用于解锁、释放内存)
- push/pop **必须成对**(底层是带 `{}` 的宏)
- 执行顺序: **后进先出(LIFO,栈)**

### 🔴 demo 5 踩坑实录 (`ch11/05_clean.c`)

**预期 + 实际现象**:
```
新线程start
资源3        ← 最后注册的最先清理
资源2
资源1        ← 最先注册的最后清理
```
**后进先出(LIFO)**,push/pop 的命名由来。

**踩的坑**(4 个):
1. **cleanup 函数签名错**:写成 `void* cleanup(void*)`,正确是 `void cleanup(void*)`(没返回值)
   → 容易和线程函数 `void *(void*)` 搞混
2. **传参类型错**:`pthread_cleanup_push(cleanup, 1)`——第二参是 `void*`,要传字符串地址 `"资源1"`,不是整数 1
3. **`%c` vs `%s`**:用 `%c` 想打字符串,应该用 `%s`
4. **变量名打错**:`tid` 写成 `pid` → 编译报 undeclared

**要点**:`pthread_cleanup_pop(0)` 的 `0` 表示"只弹注册不执行";清理由 `pthread_exit` 自动触发。

## 11.9 线程属性 ⭐(知道存在即可,面试不深究)

```c
pthread_attr_t attr;
pthread_attr_init(&attr);
pthread_attr_setstacksize(&attr, 4096);                     // 设栈大小(默认约 8MB)
pthread_attr_setdetachstate(&attr, PTHREAD_CREATE_DETACHED);// 创建即分离
pthread_create(&tid, &attr, func, NULL);
pthread_attr_destroy(&attr);
```
- 实际就能设两样:**栈大小** + **分离状态**
- 99% 情况 `attr` 传 NULL 用默认即可;嵌入式内存紧张时才调栈

## 11.11 线程与信号 ⭐(了解)

```
关键观念: 信号发给【进程】,但由其中【某一个线程】处理。
```
```c
int pthread_sigmask(int how, const sigset_t *set, sigset_t *oldset); // 设本线程信号掩码
int pthread_kill(pthread_t thread, int sig);                         // 给同进程内指定线程发信号
int pthread_sigqueue(pthread_t thread, int sig, const union sigval value); // 带数据
```
- **异步信号安全函数**(async-signal-safe):能在信号处理函数里安全调用的函数(PDF 有整张表)
- 🔵 实战套路:多线程程序通常**用一个专门线程统一处理信号**,其他线程屏蔽掉

## 🔵 全章额外补充(PDF 没讲但面试常问)

### 1. errno 为什么在多线程下安全
```
errno 是【每线程一份】的(线程独占), 一个线程的 errno 不会被别的线程覆盖。
实现上 errno 是个宏, 展开成 *__errno_location(), 每个线程返回不同地址。
```

### 2. 线程池(面试常问"了解线程池吗")
```
预先创建一批线程 → 任务来了从池里取空闲线程执行 → 用完归还,不销毁
为什么用:
  ① 省去频繁 create/destroy 的开销
  ② 控制最大并发数,防止线程爆炸
  ③ 配合任务队列削峰
```

## 一句话总结

> **线程 = 同进程内共享地址空间的执行流(调度单位):共享全局/堆/fd,独占栈/寄存器。**
> **核心 4 函数: create 创建、exit 退出、join 收尸拿返回值、detach 自动回收(二选一)。**
> **共享导致竞态 → 需要锁 → 引出第 12 章。pthread 函数返回错误码,编译加 -lpthread。**
