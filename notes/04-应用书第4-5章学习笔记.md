# 第 4 章 标准 I/O 库

## 4.1 文件 I/O vs 标准 I/O 的本质区别

| 维度 | 文件 I/O(第 2 章) | 标准 I/O(本章) |
|---|---|---|
| 谁实现的 | 内核(syscall) | C 标准库(glibc) |
| 句柄 | **fd**(int) | **FILE \***(指针) |
| 头文件 | `<unistd.h>` `<fcntl.h>` | `<stdio.h>` |
| 函数 | open/read/write/close/lseek | fopen/fread/fwrite/fclose/fseek |
| 缓冲 | 内核缓冲(动态文件) | **用户态 stdio 缓冲**(本章核心) |
| 可移植性 | Linux only | 所有支持 C 的平台 |
| 性能 | 每次都 syscall | 缓冲合并,syscall 次数少 |

### 关键洞察:标准 I/O = 文件 I/O + 用户态缓冲层

```
应用程序
   ↓ fwrite(buf, 1024, 1, fp)
stdio 用户态缓冲(默认 4KB)
   ↓ 满了或 fflush 时
write(fd, ...)  ← 文件 I/O syscall
   ↓
内核动态文件缓冲
   ↓ sync/fsync 时
磁盘
```

**两级缓冲叠加**!这是后面所有"为什么 printf 不输出 / 数据丢了"问题的根源。

## 4.2 FILE 指针(替代 fd)

```c
#include <stdio.h>
FILE *fp = fopen("file", "r");
```

- `FILE` 是一个**结构体类型**,内部包含 fd + 缓冲区 + 当前位置 + 错误标志等
- 对你来说**当黑盒用**,只通过 fopen/fclose/fread 等函数操作
- 失败返回 **`NULL`**(注意!不是 -1)

### FILE \* vs fd 的换算

```c
int fd = fileno(fp);          // FILE * → fd
FILE *fp = fdopen(fd, "r");   // fd → FILE *
```

**用途**:你拿到一个 fd(比如 dup 或 socket 返回的),想用 printf 这种格式化函数,**用 fdopen 转成 FILE \***。

## 4.3 stdin / stdout / stderr

每个进程启动时自动打开 **3 个标准流**:

| 名字 | 对应 fd | 类型 | 默认指向 |
|---|---|---|---|
| `stdin` | 0 | FILE * | 键盘 |
| `stdout` | 1 | FILE * | 显示器(**行缓冲**) |
| `stderr` | 2 | FILE * | 显示器(**无缓冲**) |

宏定义(在 `<unistd.h>`):

```c
#define STDIN_FILENO  0
#define STDOUT_FILENO 1
#define STDERR_FILENO 2
```

### ⚠ stdout 是行缓冲,stderr 是无缓冲(面试常考!)

```c
printf("Hello");           // 没 \n,留在缓冲区,屏幕看不到
fprintf(stderr, "Error");  // 立即显示
```

这就是为什么**调试信息推荐用 stderr**:不会因为缓冲被吃掉,程序崩了也能看到。

## 4.4 fopen 与 fclose

```c
FILE *fopen(const char *path, const char *mode);
int fclose(FILE *stream);
```

### mode 参数对照表(必背)

| mode | 行为 | 等价的 open flags |
|---|---|---|
| `"r"` | 只读 | `O_RDONLY` |
| `"r+"` | 读写 | `O_RDWR` |
| `"w"` | 只写,清空,不存在则创建 | `O_WRONLY \| O_CREAT \| O_TRUNC` |
| `"w+"` | 读写,清空,不存在则创建 | `O_RDWR \| O_CREAT \| O_TRUNC` |
| `"a"` | 只写,追加 | `O_WRONLY \| O_CREAT \| O_APPEND` |
| `"a+"` | 读写,追加 | `O_RDWR \| O_CREAT \| O_APPEND` |

**核心要点**:
- `"w"` 自带 `O_TRUNC`,**会清空旧文件**!别用错了
- `"a"` 自带 `O_APPEND`,多进程安全
- `"r"` 不会创建文件,文件不存在直接失败
- **fopen 不能设置文件权限**,新建文件默认权限 `0666`(实际受 umask 影响,通常变成 `0644`)

### 错误处理(返回 NULL)

```c
FILE *fp = fopen("file", "r");
if (fp == NULL) {
    perror("fopen");
    return -1;
}
```

## 4.5 fread 与 fwrite

```c
size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream);
size_t fwrite(const void *ptr, size_t size, size_t nmemb, FILE *stream);
```

### 参数解读(和 read/write 设计不同!)

- `size` × `nmemb` = 总字节数
- 读 / 写 **数据项数量**,不是字节数
- 返回值是**实际读/写的项数**(不是字节数!)

### 两种用法

```c
// 用法 1:读结构体数组(原设计意图)
struct point arr[10];
fread(arr, sizeof(struct point), 10, fp);    // 读 10 个 point

// 用法 2:当字节流(常用)
char buf[1024];
size_t n = fread(buf, 1, 1024, fp);          // 读 1024 字节,n = 实际字节数
```

**绝大多数场景用法 2**,把 `size` 设成 1,`nmemb` 设成你想读的字节数。

## 4.6 fseek / ftell / rewind(对应 lseek)

```c
int fseek(FILE *stream, long offset, int whence);
long ftell(FILE *stream);          // 返回当前偏移
void rewind(FILE *stream);          // 回到开头(= fseek(fp, 0, SEEK_SET))
```

### 与 lseek 的差异(面试常考)

| | lseek | fseek |
|---|---|---|
| 返回值 | 新的偏移量(off_t) | **0 成功,-1 失败** |
| 失败检查 | `== -1` | `!= 0` |
| 配套函数 | 没有 | ftell 查偏移、rewind 回零 |

### 用 fseek 算文件大小(经典)

```c
fseek(fp, 0, SEEK_END);
long size = ftell(fp);
rewind(fp);   // 别忘了回到开头
```

## 4.7 ferror / feof / clearerr

fread 返回值小于请求的 nmemb,**可能是错误,也可能是 EOF**。**ferror / feof 用来区分**。

```c
size_t n = fread(buf, 1, 100, fp);
if (n < 100) {
    if (ferror(fp)) {
        // 真错误
        clearerr(fp);   // 清除错误标志
    } else if (feof(fp)) {
        // 文件结束,正常
    }
}
```

## 4.8 三种缓冲模式 ⭐⭐⭐(本章最重要)

stdio 缓冲分三种:

| 模式 | 何时刷新 | 默认用在哪 |
|---|---|---|
| **全缓冲**(Full)| 缓冲区满 / fclose / fflush | 普通磁盘文件(默认 4KB) |
| **行缓冲**(Line)| 遇到 `\n` / 缓冲区满 / 刷新 | 终端关联的 stream(stdin、stdout) |
| **无缓冲**(Unbuffered)| 立即写 | stderr |

### 经典踩坑(面试必问)

```c
int main() {
    printf("Hello");      // ❌ 没 \n,留在行缓冲区
    while(1);             // 程序卡住,屏幕看不到 Hello
}
```

**Hello 在用户态 stdio 缓冲区里,没被 write 到内核,所以看不到**。修复:
- 加 `\n`:`printf("Hello\n");`(行缓冲触发刷新)
- 手动 `fflush(stdout);`
- 用 stderr:`fprintf(stderr, "Hello");`(无缓冲)

### 修改缓冲模式

```c
setvbuf(fp, NULL, _IOFBF, 0);   // 全缓冲
setvbuf(fp, NULL, _IOLBF, 0);   // 行缓冲
setvbuf(fp, NULL, _IONBF, 0);   // 无缓冲
```

### 注意:stdout 重定向到文件时变成全缓冲

```bash
./prog        # stdout 在终端,行缓冲,printf 立刻显示
./prog > out  # stdout 重定向到文件,全缓冲(4KB),内容可能延迟
```

**这就是为什么"我程序明明 printf 了,日志文件却是空的"**——重定向后行缓冲变成全缓冲。

## 4.9 格式化 I/O 家族

| 函数 | 用途 |
|---|---|
| `printf(fmt, ...)` | 到 stdout |
| `fprintf(fp, fmt, ...)` | 到 FILE * |
| `sprintf(buf, fmt, ...)` | 到字符串(**危险,不限长度**) |
| `snprintf(buf, n, fmt, ...)` | 到字符串(**安全,推荐**) |
| `scanf(fmt, ...)` | 从 stdin |
| `fscanf(fp, fmt, ...)` | 从 FILE * |
| `sscanf(s, fmt, ...)` | 从字符串 |

### 常用格式符

| 符号 | 类型 |
|---|---|
| `%d` `%i` | int |
| `%u` | unsigned int |
| `%ld` | long |
| `%lld` | long long |
| `%zd` | ssize_t(read 返回) |
| `%zu` | size_t |
| `%x` `%X` | 十六进制 |
| `%o` | 八进制 |
| `%c` | char |
| `%s` | char *(字符串) |
| `%p` | 指针 |
| `%f` | float / double |
| `%.3f` | 3 位小数 |
| `%5d` | 至少 5 宽度 |

### 安全建议:**永远不要用 `sprintf` 和 `scanf("%s", ...)`**

```c
char buf[10];
sprintf(buf, "%s", input);     // ❌ 缓冲区溢出风险
scanf("%s", buf);              // ❌ 不限长度

snprintf(buf, sizeof(buf), "%s", input);   // ✅ 限制长度
scanf("%9s", buf);                         // ✅ 限制 9 字符 + \0
```

## 4.10 标准 I/O 和文件 I/O 混合(踩坑警告)

```c
FILE *fp = fopen("file", "r");
int fd = fileno(fp);
read(fd, buf, 100);    // 直接绕过 stdio 缓冲读
fread(buf2, 1, 100, fp);  // 用 stdio 读,但缓冲混乱了
```

**永远不要在同一个 FILE \* 上混用 fread/fwrite 和 read/write**——会导致 stdio 缓冲和实际文件位置不一致,产生玄学 bug。

# 第 5 章 文件属性与目录

## 5.1 七种文件类型(必背)

Linux 万物皆文件,分 7 种:

| 类型 | 缩写 | ls -l 首字符 | 例子 |
|---|---|---|---|
| 普通文件 | regular | `-` | hello.c |
| 目录 | directory | `d` | /home |
| **字符设备** | char device | `c` | /dev/tty, /dev/random |
| **块设备** | block device | `b` | /dev/sda, /dev/mmcblk0 |
| 符号链接 | symbolic link | `l` | ln -s 创建的 |
| 管道 | FIFO | `p` | mkfifo 创建的 |
| 套接字 | socket | `s` | /tmp/mysql.sock |

**特别注意字符设备 / 块设备**——这是你后面学驱动开发的核心:
- 字符设备 = **按字节流操作**(LED、按键、串口、I2C 传感器)
- 块设备 = **按块操作,有缓存**(磁盘、SD 卡、eMMC)

后面驱动指南第 40 章直接就叫"**字符设备驱动开发**",和这里对应。

## 5.2 stat 系列函数 ⭐⭐⭐

```c
#include <sys/stat.h>
int stat(const char *path, struct stat *buf);   // 按路径查
int fstat(int fd, struct stat *buf);             // 按 fd 查
int lstat(const char *path, struct stat *buf);   // 查链接本身,不跟随
```

### stat 结构体(核心)

```c
struct stat {
    dev_t     st_dev;       // 设备 ID
    ino_t     st_ino;       // inode 编号
    mode_t    st_mode;      // 文件类型 + 权限位
    nlink_t   st_nlink;     // 硬链接数
    uid_t     st_uid;       // 所有者用户 ID
    gid_t     st_gid;       // 所有者组 ID
    dev_t     st_rdev;      // 如果是设备文件,设备号
    off_t     st_size;      // 文件大小(字节)
    blksize_t st_blksize;   // 文件系统 IO 块大小
    blkcnt_t  st_blocks;    // 占用块数(512 字节/块)
    time_t    st_atime;     // 最后访问时间
    time_t    st_mtime;     // 最后修改时间(内容)
    time_t    st_ctime;     // 最后状态变更时间(权限/所有者)
};
```

**用 stat 算文件大小**(比 lseek + SEEK_END 优雅):

```c
struct stat st;
stat("file", &st);
printf("size = %ld\n", (long)st.st_size);
```

## 5.3 文件类型判断宏(用 st_mode)

```c
struct stat st;
stat("/dev/sda", &st);

if (S_ISREG(st.st_mode))  /* 普通文件 */;
if (S_ISDIR(st.st_mode))  /* 目录 */;
if (S_ISCHR(st.st_mode))  /* 字符设备 */;
if (S_ISBLK(st.st_mode))  /* 块设备 */;
if (S_ISFIFO(st.st_mode)) /* 管道 */;
if (S_ISLNK(st.st_mode))  /* 符号链接 */;
if (S_ISSOCK(st.st_mode)) /* 套接字 */;
```

**面试灵魂问题**:"判断一个文件是不是普通文件?"
**答**:`S_ISREG(st.st_mode)`。

## 5.4 权限位与 chmod

st_mode 低 12 位是权限,3 位一组:

| 位 | 用户类型 | 含义 |
|---|---|---|
| bits 8-6 | owner | rwx |
| bits 5-3 | group | rwx |
| bits 2-0 | other | rwx |

**八进制表示**:
- `0644` = `rw-r--r--`(常见普通文件)
- `0755` = `rwxr-xr-x`(可执行文件、目录)
- `0700` = `rwx------`(私人文件)

### chmod / fchmod

```c
int chmod(const char *path, mode_t mode);
int fchmod(int fd, mode_t mode);

chmod("file", 0755);
```

### umask(掩码)

新建文件时,**实际权限 = mode & ~umask**。

- 默认 umask 通常 `0022`
- fopen 用 mode "w" 想给 `0666`,被 umask 0022 屏蔽后 = `0644`
- 想完全控制权限,**先 `umask(0)`**

## 5.5 文件所有者(uid / gid)

```c
int chown(const char *path, uid_t owner, gid_t group);
int fchown(int fd, uid_t owner, gid_t group);
int lchown(const char *path, uid_t owner, gid_t group);
```

- 改文件所有者(通常 root 才能改)
- 嵌入式实际项目里用得不多,知道有这函数即可

## 5.6 文件时间属性

| 时间 | 字段 | 何时更新 |
|---|---|---|
| **atime**(access)| st_atime | 文件被**读取**时 |
| **mtime**(modify)| st_mtime | 文件**内容修改**时 |
| **ctime**(change)| st_ctime | **元数据**(权限/所有者/链接数)改变时 |

**注意 ctime 不是"创建时间"**!Linux 标准接口没有"创建时间"概念(ext4 新版本支持 btime 但不通用)。

### 用 utime() 修改时间

```c
#include <utime.h>
struct utimbuf {
    time_t actime;   // 设置 atime
    time_t modtime;  // 设置 mtime
};
utime("file", &times);
```

## 5.7 硬链接 vs 符号链接 ⭐⭐⭐(高频面试题)

### 创建

```c
int link(const char *oldpath, const char *newpath);       // 硬链接
int symlink(const char *target, const char *linkpath);    // 符号链接(软链接)
```

命令行:`ln file hardlink` / `ln -s file softlink`

### 本质对比(必背!)

| 维度 | 硬链接 | 符号链接 |
|---|---|---|
| **inode** | **同一个 inode** | **新 inode** |
| 跨文件系统 | ❌ 不行 | ✅ 可以 |
| 链到目录 | ❌ 普通用户不行 | ✅ 可以 |
| 源文件删除后 | **链接仍可用** | **变成悬空(dangling)** |
| 文件大小 | 等于源文件 | 等于路径字符串长度 |
| 实现 | 多个目录项指向同一 inode | 单独文件,内容是目标路径字符串 |
| 类比 | 文件的"别名" | Windows 快捷方式 |

### 硬链接演示

```bash
echo "hello" > file
ln file hardlink

ls -li file hardlink
# 看到两个文件 inode 号一样,链接数都是 2

rm file
cat hardlink   # 还能读到 "hello"!因为 inode 还在
```

### 符号链接演示

```bash
ln -s file softlink
ls -li file softlink
# 看到 inode 不同,softlink 是 l 类型

rm file
cat softlink   # 报错:No such file or directory
```

### 嵌入式应用

- `/dev` 下大量符号链接(`/dev/stdin → /proc/self/fd/0`)
- `/etc/rc.d` 下用符号链接管理启动脚本顺序
- 你后面学 Buildroot 会大量遇到符号链接

## 5.8 读链接 readlink

```c
ssize_t readlink(const char *path, char *buf, size_t bufsiz);
```

**读出符号链接指向的路径字符串**(不解析、不跟随):

```c
char buf[256];
ssize_t n = readlink("/dev/stdin", buf, sizeof(buf)-1);
buf[n] = '\0';
printf("%s\n", buf);   // 输出 /proc/self/fd/0
```

## 5.9 删除 unlink、重命名 rename

```c
int unlink(const char *path);                      // 删除文件
int rename(const char *oldpath, const char *newpath);  // 重命名/移动
```

**unlink 的本质**:
- **减少 inode 的链接数**
- 当链接数 = 0 **且** 没有进程持有这个文件的 fd → 真删
- 这就是为什么 "**rm 一个被进程打开的日志文件,日志还在写,直到进程关闭 fd**"

### 经典骚操作:打开后立即 unlink

```c
int fd = open("/tmp/tmpfile", O_RDWR | O_CREAT, 0600);
unlink("/tmp/tmpfile");   // 立刻 unlink,但文件还能用
// ... 程序运行期间用 fd 读写 ...
close(fd);  // 这时真删了
```

**用途**:**临时文件**,无论程序怎么挂,文件都会被清理(因为 unlink 已经做了)。

## 5.10 目录操作

### 创建 / 删除目录

```c
int mkdir(const char *path, mode_t mode);   // 创建目录(权限受 umask 影响)
int rmdir(const char *path);                // 删除空目录
```

### 遍历目录(opendir / readdir / closedir)⭐

```c
#include <dirent.h>
DIR *opendir(const char *name);
struct dirent *readdir(DIR *dirp);
int closedir(DIR *dirp);
```

`struct dirent` 含 `d_name`(文件名) + `d_type`(类型)。

### 经典:用代码实现 `ls`

```c
DIR *dir = opendir(".");
if (dir == NULL) { perror("opendir"); return -1; }

struct dirent *entry;
while ((entry = readdir(dir)) != NULL) {
    printf("%s\n", entry->d_name);
}
closedir(dir);
```

**面试可能让你手写这段**——可以放进 GitHub 仓库。

### 切换 / 获取工作目录

```c
int chdir(const char *path);             // cd
char *getcwd(char *buf, size_t size);    // pwd
```

## 我对这两章的整体看法

**两章相对工具性,不像第 3 章那么"硬核"**。但有几个点对你后面学习极其关键:

### 通向驱动开发的桥梁

1. **字符设备 / 块设备**(5.1 节)→ 直接对应驱动书第 40 章
2. **stat 结构体**(5.2 节)→ 后面写驱动的 sysfs 节点属性常用
3. **特殊文件**(/dev/、/proc/、/sys/)是 Linux 驱动开发的舞台

### 通向并发编程的桥梁

1. **stdio 缓冲模式**(4.8 节)→ 后面学多进程 / 多线程时,**fork 会复制缓冲区,导致输出重复**
2. **stderr 无缓冲特性**→ 之后你写守护进程会爱上它

### 实际工作里用得最多的是?

- `stat()` 算文件大小、判断类型(写代码每周都会用)
- `opendir / readdir` 遍历文件(任何"批量处理文件"的工具)
- `snprintf` 替代 sprintf(代码安全)
- `fprintf(stderr, ...)` 打调试信息

## 关键代码模板

### 模板 1:用 stat 算文件大小

```c
struct stat st;
if (stat("file", &st) == 0) {
    printf("size = %ld\n", (long)st.st_size);
}
```

### 模板 2:判断是否是普通文件

```c
struct stat st;
stat(path, &st);
if (S_ISREG(st.st_mode)) {
    /* 是普通文件 */
}
```

### 模板 3:遍历目录

```c
DIR *dir = opendir(".");
struct dirent *entry;
while ((entry = readdir(dir)) != NULL) {
    if (strcmp(entry->d_name, ".") == 0 ||
        strcmp(entry->d_name, "..") == 0)
        continue;   // 跳过 . 和 ..
    printf("%s\n", entry->d_name);
}
closedir(dir);
```

### 模板 4:自删除临时文件

```c
int fd = open("/tmp/data", O_RDWR | O_CREAT | O_EXCL, 0600);
unlink("/tmp/data");   // 程序运行期间文件还在,close 后自动消失
/* ... 用 fd 读写 ... */
close(fd);
```

### 模板 5:格式化字符串(安全版)

```c
char buf[64];
snprintf(buf, sizeof(buf), "log_%d.txt", id);
```

## 踩坑预警(后面学到时回来看)

### 1. fopen 的 "w" 会清空文件
和 open 一样,`"w"` 隐含 `O_TRUNC`,别用错了。

### 2. stdout 重定向后变成全缓冲
`./prog > log` 后,printf 内容可能延迟到 4KB 才写入文件,**程序崩了 log 是空的**。**关键路径用 `fflush(stdout)` 或 `setvbuf` 改无缓冲**。

### 3. fork 后子进程继承缓冲区
父进程 printf 没刷新,fork 后**父子都会再输出一遍**。守护进程要 fflush 后再 fork。

### 4. 硬链接不能跨文件系统
NFS、tmpfs、不同分区之间不能硬链接,会报 `EXDEV`(Cross-device link)。

### 5. readdir 返回的 `.` 和 `..` 要跳过
否则遍历时会无限递归。

### 6. ctime ≠ 创建时间
ctime = change time(状态变更),Linux 没有标准创建时间字段。

## 一句话总结

> **第 4 章:FILE \* 是带缓冲的 fd 包装。缓冲三模式(全 / 行 / 无),理解了就不会"printf 不输出"。**
>
> **第 5 章:stat 拿元数据,inode 决定身份,硬链接共享 inode,符号链接是路径字符串。**
