## 6.1 字符串 I/O

| 函数 | 功能 | 安全? |
|---|---|---|
| `gets(buf)` | 从 stdin 读一行 | ❌ **绝对不能用!**C11 已删除,缓冲区溢出经典 |
| `fgets(buf, n, fp)` | 读一行到 buf,最多 n-1 个字符 | ✅ 推荐 |
| `puts(s)` | 输出到 stdout + 自动 `\n` | ✅ |
| `fputs(s, fp)` | 输出到指定流,**不加 \n** | ✅ |
| `getchar()` / `putchar()` | 单字符 | ✅ |

### 标准读一行的代码

```c
char buf[256];
if (fgets(buf, sizeof(buf), stdin) == NULL) {
    // EOF 或错误
}
// 注意:fgets 会保留行尾的 \n,需要手动去掉
buf[strcspn(buf, "\n")] = '\0';
```

## 6.2 字符串长度

```c
#include <string.h>
size_t strlen(const char *s);
```

### ⚠ sizeof vs strlen(面试常考)

```c
char s[10] = "abc";
sizeof(s);    // = 10(数组的内存大小,包括 \0 和未用空间)
strlen(s);    // = 3(到 \0 之前的字符数,不含 \0)

char *p = "abc";
sizeof(p);    // = 8(指针大小,64 位系统)
strlen(p);    // = 3
```

**核心区别**:
- `sizeof` 是编译期算的,看类型
- `strlen` 是运行期算的,扫到 `\0` 为止

## 6.3 字符串拷贝

| 函数 | 用法 | 安全? |
|---|---|---|
| `strcpy(dst, src)` | 拷贝整串 | ❌ **不限长度,缓冲区溢出** |
| `strncpy(dst, src, n)` | 最多拷 n 字节 | ⚠️ **可能不加 \0,陷阱** |
| `strlcpy(dst, src, n)` | BSD 版,保证 \0 | ✅ Linux 默认没这函数 |
| `snprintf(dst, n, "%s", src)` | 万能替代 | ✅ **最推荐** |
| `memcpy(dst, src, n)` | 按字节拷贝(可含 \0) | ✅ |
| `memmove(dst, src, n)` | 处理重叠区域 | ✅ |

### strncpy 的陷阱(必踩)

```c
char dst[5];
strncpy(dst, "Hello", 5);   // ⚠️ 拷了 5 字节,**没有 \0**!
printf("%s\n", dst);         // 后面接着读内存,可能崩
```

**修复**:

```c
strncpy(dst, "Hello", sizeof(dst) - 1);
dst[sizeof(dst) - 1] = '\0';   // 强制结尾
```

### 推荐用法:snprintf 大法

```c
char dst[64];
snprintf(dst, sizeof(dst), "%s", src);   // 永远安全
```

## 6.4 字符串拼接

```c
char *strcat(char *dst, const char *src);          // ❌ 不限长度
char *strncat(char *dst, const char *src, size_t n); // ⚠️ n 是 src 的字节数,不是 dst 总长!
```

### strncat 的陷阱

```c
char dst[10] = "abc";
strncat(dst, "1234567890", 5);
// 这个 5 是从 src 里取 5 字节,不是限制 dst 总长 5
// 结果:dst = "abc12345" + \0,长度 8(安全)
// 但如果 dst 已经 9 字节,再 strncat 还是会溢出
```

**推荐替代**:

```c
snprintf(dst + strlen(dst), sizeof(dst) - strlen(dst), "%s", src);
// 或者改用临时变量分两步算
```

## 6.5 字符串比较

```c
int strcmp(const char *s1, const char *s2);          // 全比较
int strncmp(const char *s1, const char *s2, size_t n); // 前 n 字符比较
int strcasecmp(...);                                  // 不区分大小写(GNU 扩展)
int memcmp(const void *s1, const void *s2, size_t n); // 内存比较(含 \0)
```

### 返回值

- `< 0`:s1 小
- `= 0`:相等
- `> 0`:s1 大

### 常见误用

```c
if (strcmp(s, "hello"))  // ❌ 这是"不相等"的意思!
if (strcmp(s, "hello") == 0)  // ✅ 相等
```

## 6.6 字符串查找

| 函数 | 功能 |
|---|---|
| `strchr(s, c)` | 找字符 c 第一次出现的位置(指针) |
| `strrchr(s, c)` | 找字符 c **最后一次**出现 |
| `strstr(s, sub)` | 找子串 sub 的位置 |
| `strpbrk(s, set)` | 找 set 里**任一字符**的位置 |
| `strspn(s, set)` | 从头开始,**全是 set 里字符**的长度 |
| `strcspn(s, set)` | 从头开始,**不含 set 里字符**的长度(常用!) |
| `strtok(s, delim)` | **分割字符串**(会修改原串,有线程安全问题) |
| `strtok_r(s, delim, &saveptr)` | 线程安全版 |

### strtok 陷阱

```c
char s[] = "a,b,c";              // ⚠️ 必须是数组,不是 char *
char *tok = strtok(s, ",");
while (tok) {
    printf("%s\n", tok);
    tok = strtok(NULL, ",");     // 后续用 NULL
}
// 输出:a / b / c
// 副作用:原字符串 s 被改成 "a\0b\0c\0"
```

**线程不安全**(用全局状态),多线程用 `strtok_r`。

## 6.7 字符串与数字互转

### 字符串 → 数字

| 函数 | 用法 | 安全? |
|---|---|---|
| `atoi(s)` | 转 int | ⚠️ 无错误检查,失败返回 0(和合法 0 没法区分) |
| `atol(s)` | 转 long | 同上 |
| `atof(s)` | 转 double | 同上 |
| `strtol(s, &end, base)` | 转 long,**有错误检查 + 支持进制** | ✅ **推荐** |
| `strtoul(s, &end, base)` | 转 unsigned long | ✅ |
| `strtod(s, &end)` | 转 double | ✅ |

### strtol 推荐用法

```c
#include <stdlib.h>
#include <errno.h>

char *end;
errno = 0;
long val = strtol("123abc", &end, 10);
// end 指向 'a'(第一个无法解析的位置)
// 如果 end == s,说明根本没解析出数字

if (end == s) /* 没解析出 */;
if (errno == ERANGE) /* 溢出 */;
if (*end != '\0') /* 部分解析,后面还有垃圾 */;

// 解析十六进制
long hex = strtol("0xff", &end, 16);  // 或 base = 0 自动识别 0x / 0
```

### 数字 → 字符串

C 标准库**没有 itoa**(Windows 才有)。**用 snprintf**:

```c
char buf[16];
snprintf(buf, sizeof(buf), "%d", 42);       // 十进制
snprintf(buf, sizeof(buf), "0x%x", 255);    // 十六进制
snprintf(buf, sizeof(buf), "%.3f", 3.14);   // 浮点
```

## 6.8 内存操作(mem 家族)

| 函数 | 功能 | 用途 |
|---|---|---|
| `memset(p, c, n)` | 把 n 字节填成 c | 初始化缓冲区(`memset(buf, 0, sizeof(buf))`) |
| `memcpy(dst, src, n)` | 拷贝 n 字节 | 不带 \0 的内存块复制 |
| `memmove(dst, src, n)` | 拷贝 n 字节,**处理重叠** | dst 和 src 区域重叠时用这个 |
| `memcmp(p1, p2, n)` | 比较 n 字节 | 结构体比较、二进制比较 |
| `memchr(p, c, n)` | 在 n 字节里找 c | 二进制数据里找特定字节 |

### mem 家族 vs str 家族

| | str* | mem* |
|---|---|---|
| 终止判断 | 遇到 `\0` 停 | 按 n 字节算 |
| 适用 | 字符串(以 \0 结尾) | 任意二进制数据 |
| 速度 | 每次都要扫 \0,慢 | 直接按字节,快 |

**铁律**:
- 处理**真正的字符串**用 `str*`
- 处理**二进制 / 结构体 / 含 \0 的数据**用 `mem*`

## 6.9 可变参数 va_list

用来写自定义的 printf 类函数。

```c
#include <stdarg.h>

void my_log(const char *fmt, ...) {
    va_list args;
    va_start(args, fmt);
    vprintf(fmt, args);    // 传给 vprintf
    va_end(args);
}

my_log("count = %d, name = %s\n", 42, "hello");
```

## 6.10 / 6.11 正则表达式(精简版)

### 啥时候用正则?

**嵌入式里 99% 不用**。能用 `strstr` / `strchr` / `sscanf` 解决的就别上正则。

### 基础语法(背 10 个符号)

| 符号 | 含义 |
|---|---|
| `.` | 任意一个字符 |
| `*` | 前一字符 0 次或多次 |
| `+` | 前一字符 1 次或多次 |
| `?` | 前一字符 0 或 1 次 |
| `^` | 行首 |
| `$` | 行尾 |
| `[abc]` | a 或 b 或 c |
| `[0-9]` | 范围 |
| `[^abc]` | 不是 abc |
| `\|` | 或(如 `cat\|dog`) |

### POSIX C API 三步骨架

```c
#include <regex.h>

regex_t reg;
regcomp(&reg, "pattern", REG_EXTENDED);   // 1. 编译
if (regexec(&reg, "input", 0, NULL, 0) == 0) {   // 2. 匹配
    // 匹配成功
}
regfree(&reg);                            // 3. 释放
```

## 危险函数 → 安全替代(铁律必背)

| 永远别用 | 推荐替代 |
|---|---|
| `gets(buf)` | `fgets(buf, n, stdin)` |
| `strcpy(dst, src)` | `snprintf(dst, n, "%s", src)` |
| `strcat(dst, src)` | `snprintf` 或自己算长度 |
| `sprintf(buf, ...)` | `snprintf(buf, n, ...)` |
| `atoi(s)` | `strtol(s, &end, 10)` + 错误检查 |
| `strtok(s, ...)` | 多线程下用 `strtok_r` |

**面试官如果看到你写 strcpy 不写 strncpy,直接减分**。

```c
char line[256];
while (fgets(line, sizeof(line), fp)) {
    line[strcspn(line, "\n")] = '\0';

    char *eq = strchr(line, '=');
    if (eq) {
        *eq = '\0';
        char *key = line;
        char *value = eq + 1;
        // 处理 key = value
    }
}
```
