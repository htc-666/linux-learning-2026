## 第 1 章 应用编程概念

### 1.1 核心概念:系统调用 vs 库函数

| 维度 | 系统调用 (syscall) | C 库函数 (libc) |
|---|---|---|
| 谁实现的 | 内核 | C 标准库(glibc) |
| 例子 | open / read / write / close | fopen / fread / fwrite / printf |
| 调用代价 | 高(陷入内核态) | 低(纯用户态) |
| 是否带缓冲 | 否,直接和内核交互 | 是,有 stdio 缓冲 |
| 关系 | 底层 | 在系统调用之上封装 |

**关键洞察**:`printf` 内部最终调用 `write(1, ...)` 系统调用,但中间有一层 stdio 行缓冲。这就是为什么 "`printf` 不带 `\n` 可能不立即输出"。

### 1.2 开发环境

- **不用 Eclipse**,用 VSCode + 终端 gcc 即可
- 基本编译:`gcc hello.c -o hello`
- **强烈推荐 -Wall**:`gcc -Wall hello.c -o hello`(打开所有警告)

## 第 2 章 文件 I/O 基础

### 2.1 文件描述符 (file descriptor, fd)

- **本质**:一个整数,内核用它索引进程的"打开文件表"
- **进程默认占用**:
  - `0` = stdin(标准输入)
  - `1` = stdout(标准输出)
  - `2` = stderr(标准错误)
- **新打开的文件**:从 3 开始分配
- **失败信号**:open / read / write / lseek 失败统一返回 `-1`

### 2.2 open() 系统调用

```c
#include <fcntl.h>
int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);
```

#### 常用 flags

| Flag | 含义 |
|---|---|
| O_RDONLY | 只读 |
| O_WRONLY | 只写 |
| O_RDWR | 读写 |
| O_CREAT | 不存在则创建(需配 mode) |
| O_TRUNC | 已存在则清空 |
| O_APPEND | 写入时追加到末尾(原子) |
| O_EXCL | **必须配合 O_CREAT**:已存在则失败 |

#### mode 参数(权限)

- 只在 O_CREAT 时需要
- 八进制:`0744` = `rwxr--r--`
  - 7 = rwx(所有者)
  - 4 = r--(同组)
  - 4 = r--(其他)
- 等价宏:`S_IRWXU | S_IRGRP | S_IROTH` = 0744

### 2.3 read() 系统调用

```c
ssize_t read(int fd, void *buf, size_t count);
```

- 返回值:**实际读到的字节数**(可能 < count,叫"短读")
- 返回 0:到达文件末尾
- 返回 -1:错误,errno 被设置

### 2.4 write() 系统调用

```c
ssize_t write(int fd, const void *buf, size_t count);
```

- 返回值:实际写入字节数
- **配合 read 的正确用法**:

```c
ssize_t n = read(src, buf, sizeof(buf));
write(dst, buf, n);    // 写 n,不是 sizeof(buf)
```

### 2.5 lseek() 系统调用

```c
off_t lseek(int fd, off_t offset, int whence);
```

#### whence 三个值

| whence | 含义 |
|---|---|
| SEEK_SET | 从文件开头偏移 offset |
| SEEK_CUR | 从当前位置偏移 offset |
| SEEK_END | 从文件末尾偏移 offset |

#### 妙用:计算文件大小

```c
off_t size = lseek(fd, 0, SEEK_END);
// size 就是文件大小
```

### 2.6 close()

```c
int close(int fd);
```

- 谁 open,谁 close
- 程序退出系统自动 close,但显式 close 是好习惯

## 第 2 章 练习题完成记录

### 题目 (1) 偏移复制

从 src_file 偏移 500 处读 1024 字节,写到 dest_file 开头(权限 0744)

```c
char buf[1024] = {0};
int fd1 = open("src_file", O_RDONLY);
int fd2 = open("dest_file", O_WRONLY | O_CREAT | O_TRUNC, 0744);
lseek(fd1, 500, SEEK_SET);
ssize_t n = read(fd1, buf, sizeof(buf));
write(fd2, buf, n);
close(fd1); close(fd2);
```

验证:

```bash
yes "ABCDEFG" | head -c 2048 > src_file
./ex2_1
xxd -s 500 -l 32 src_file
xxd -l 32 dest_file   # 应该一样
```

### 题目 (2) 文件存在判断

```c
#include <errno.h>
int fd = open("test_file", O_RDONLY);
if (fd == -1) {
    if (errno == ENOENT) {
        printf("test_file does not exist\n");
    } else {
        perror("open test_file");
    }
}
```

注意:
- 不要用 O_EXCL(必须配 O_CREAT)
- 成功路径用 printf,不是 perror
- 严谨写法区分 errno 不同值

### 题目 (3) 二进制填充

```c
char buf[1024];
int fd = open("new_file", O_WRONLY | O_CREAT | O_TRUNC, 0744);
memset(buf, 0x00, 1024);
write(fd, buf, 1024);
memset(buf, 0xFF, 1024);
write(fd, buf, 1024);
close(fd);
```

注意:必须用 char,不能用 int(int 是 4 字节)

### 题目 (4) lseek 算文件大小

```c
int fd = open("test_file", O_RDONLY);
off_t size = lseek(fd, 0, SEEK_END);
printf("size = %ld\n", (long)size);
close(fd);
```

## 踩坑记录(13 个真实问题)

### 1. printf 引号位置错误

```c
printf("fd = %d\n, fd");      // 闭合引号在 fd 后,变量没传给 %d
printf("fd = %d\n", fd);      // 闭合在 \n 后,fd 是参数
```

症状:输出 `fd = 1322325844`(栈上垃圾值)

### 2. 文件未保存就编译

症状:`undefined reference to main`(链接器找不到 main)
原因:VSCode 编辑器里改了代码但没 Ctrl+S,磁盘上是旧文件
解决:settings.json 加 `"files.autoSave": "afterDelay"`

### 3. int buf vs char buf

```c
int buf[1024];      // 占 4096 字节(每个 int 4 字节)
char buf[1024];     // 占 1024 字节
```

**铁律**:字节级操作只用 char,不用 int

### 4. 聚合初始化器误用

```c
char buf[1024] = {0};     // 声明时可以
buf[1024] = {1};          // 语法错!后续赋值不能用 {}
```

正确做法:用 memset 填充

```c
memset(buf, 0xFF, 1024);
```

### 5. O_EXCL 不能单独用

```c
open("file", O_EXCL);              // undefined behavior
open("file", O_CREAT | O_EXCL);    // 正确,原子创建,已存在则失败
```

### 6. perror 只用在错误路径

```c
if (fd == -1) perror("open");      // 错误路径,perror 输出 errno 含义
printf("ok\n");                    // 成功路径用 printf
```

### 7. lseek 返回值类型

```c
int n = lseek(fd, 0, SEEK_END);     // 大文件溢出
off_t n = lseek(fd, 0, SEEK_END);   // 支持 64 位偏移
printf("%ld\n", (long)n);
```

### 8. 短读 (short read)

- read 返回值 ≤ count,不保证读满
- write 必须用 read 返回值作为字节数

```c
ssize_t n = read(fd1, buf, sizeof(buf));
write(fd2, buf, n);    // n,不是 sizeof
```

### 9. 忘了 O_TRUNC

症状:每次跑程序,旧文件后段数据残留,以为代码 bug
解决:写新文件几乎永远要 `O_WRONLY | O_CREAT | O_TRUNC`

### 10. errno 用法

```c
#include <errno.h>
if (open(...) == -1) {
    if (errno == ENOENT) /* 文件不存在 */;
    else if (errno == EACCES) /* 没权限 */;
}
```

常用 errno:
- `ENOENT` — 不存在
- `EACCES` — 没权限
- `EISDIR` — 是目录
- `EEXIST` — 已存在(O_EXCL 时)

### 11. 编译警告不能忽视

- 一定加 -Wall
- format 警告 = 90% 是 bug(如 %d 没传 int)
- unused variable 警告 = 写漏了代码

### 12. 生成测试文件

```bash
dd if=/dev/urandom of=src_file bs=1 count=2048      # 随机内容
yes "ABCDEFG" | head -c 2048 > src_file             # 可读内容
```

### 13. xxd 验证二进制

```bash
xxd file              # 全文件
xxd -s 500 -l 32 file # 从偏移 500 看 32 字节
xxd -l 32 file        # 前 32 字节
```

## 关键编程习惯总结

| 习惯 | 为什么 |
|---|---|
| 编译加 -Wall | 90% bug 编译时能抓到 |
| 错误用 perror | 自动带 errno 描述 |
| 写文件加 O_TRUNC | 保证文件干净 |
| write 用 read 返回值 | 避免短读多写垃圾 |
| 谁 open 谁 close | 资源管理 |
| 字节操作用 char | 类型对齐 |
| 大小用 off_t / ssize_t | 支持大文件 |
| 错误消息英文 | 国际化 + grep 友好 |

## 学习方法论(从抄代码到独立写)

反模式(最初的做法):
- 看书 → 抄完整代码 → 改名字跑通 → 学到了"个寂寞"

正确做法:
- 拆成最小可理解程序(每个 5-10 行)
- 每个程序只验证一个 syscall
- 写完一个停 5 分钟消化
- 累积到能"伪代码 → C 翻译"
- 卡住时只看最小提示,不抄完整答案

---

## 一句话总结

> **fd 是个 int,-1 是失败,errno 是细节,close 收尾。**
>
> 用 -Wall 编译,用 perror 报错,用 xxd 验证。
